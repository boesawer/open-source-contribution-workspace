## Workflow

I try to keep each contribution small and easy to review:

1. Read the issue and related discussion.
2. Check whether someone is already working on it.
3. Inspect the relevant files locally.
4. Make the smallest useful change.
5. Run the most relevant checks available.
6. Open a pull request with a clear summary and testing notes.

## Notes

This is a practical contribution workspace. Not every issue becomes a pull request, and not every pull request gets merged. The aim is steady progress, better judgment, and cleaner contributions over time.
