# Open Source Contribution Workspace

This repository tracks my open-source contribution practice: finding approachable issues, making focused changes, and documenting the workflow I use from issue review to pull request.

The goal is simple: become a more consistent contributor by shipping small, useful improvements and learning from maintainer feedback.

## Current Focus

- JavaScript and TypeScript projects
- React and Node.js applications
- Documentation improvements
- Small UI and UX fixes
- Backend quality-of-life improvements
- Clear, reviewable pull requests

## Contributions

See [CONTRIBUTIONS.md](./CONTRIBUTIONS.md) for submitted pull requests.

## Workflow

I try to keep each contribution small and easy to review:

1. Read the issue and related discussion.
2. Check whether someone is already working on it.
3. Inspect the relevant files locally.
4. Make the smallest useful change.
5. Run the most relevant checks available.
6. Open a pull request with a clear summary and testing notes.

## Notes

This is a learning-focused workspace. Not every issue becomes a pull request, and not every pull request gets merged. The aim is steady progress, better judgment, and cleaner contributions over time.
